# Initial Product Backlog : Smart Community Issue Reporting System
## Scrum Master Take Note
## These are initial backlogs that need to be done in sprint cycles.
| Priority | User Story ID | User Story
| :--- | :--- | :--- |
| **High** | 001 | As a resident, I want to have the ability to create an account, where I can sign up and login, so that my reports are saved. |
| **High** | 002 | As a resident, I want to be able to upload photos or images as supporting documents for my issue reported. |
| **High** | 003 | As a resident, I want to be able to add my location so that the repair team can track my location to fix the issue i reported. |
| **High** | 004 | As an admin, I want to be able to see all the issues that residents have reported. |
| **Medium** | 005 | As an admin, I want a function that allows to put priorities of the issues to be sorted. |
| **Medium** | 006 | As a resident, I want to be able to track my report status like (Reported, In progress, Finished) to be sorted in stages, so that I know that my issues are being fixed.
