Project Description:
This project is designed to solve a real-world friction point aligned with Sustainable Development Goal 11: Sustainable Cities and Communities. Our goal is to create a prototype that makes urban living smoother and more inclusive through smart technology. This project aims to help communities to solve urban issues and deal directly and easier with the authorities in charge.



Overview of SDLC Work:
1. Planning
Product Backlog & Roles:
We delegated our tasks according to each of our roles, like 1 person is in charge of a certain phase in SDLC. We collected user requirements for our Product Backlog using Google Forms. We used Trello for our Product and Sprint Backlogs to track tasks through "To Do," "Doing," and "Done".

2. Design
We mainly used Visual Paradigm CE tool to create Use Case and Class Diagrams that were essential to represent our system's functional interactions. To design UI/UX designs, we used Figma to create wireframes to guide the early prototype build. For UI sketches, we manually drew the sketches by hand.

3. Development
For the implementation part, we used Agile methods, such as iterative sprints, (Sprint 1 and 2) to build our prototype. Our Scrum Master helped organizing the tasks for the members to complete each sprint. Not only that, for version control, we utilized GitHub by committing regular meaningful messages to document our progress on the SDLC progress. We uploaded our screenshots of our early prototypes in GitHub too.

4. Testing & Evaluation
For testing phase, we included user-feedback to improve our prototype. We created testing instruction manual as a guide to test our prototype. We documented user's response by using google forms. We also utilized bug reports of our prototype. This really helped us to identify errors in our system and improve them easily.
